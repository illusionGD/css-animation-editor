## 记忆库管理（Memory Bank Management）

你被允许读取和写入 `.cline/memory/` 目录下的文件。

当你发现以下情况时：
- 稳定且长期成立的项目事实
- 重复出现的用户偏好
- 已明确且不会频繁推翻的架构决策
- 已确认、值得在未来避免再次发生的错误

你【必须】：
1. 提议更新对应的记忆文件
2. 明确说明为什么这些信息具有「长期价值」
3. 在写入之前向用户请求确认
4. 使用中文回答

【不要】存储以下内容：
- 进行中的任务或 TODO 状态
- 临时方案、探索性实现
- 一次性实验代码
- 尚未验证或可能被推翻的想法


## 规则自我改进（Rules Self-Improvement）

如果你注意到以下情况：
- 某条指令被反复要求
- 某个约束能够明显提升输出质量
- 某种工作流程模式在多个任务中持续有效

你可以提议更新本规则文件。

你【必须】：
- 清楚说明引入该规则的实际收益
- 给出精确、可审查的 diff
- 在实际编辑前等待用户的明确批准
- vue结构顺序：template-script-style


## 记忆触发条件（Memory Triggers）

在以下情况下，应当考虑更新记忆库：
- 相同的指令出现 3 次或以上
- 用户使用了 “always / never / from now on” 等明确长期意图的表述
- 某个决策会影响多个模块、文件或未来功能扩展
- 完成了具有「架构意义」的组件或模块（例如：核心 Store、渲染策略、时间轴模型），需要更新文件路径+描述


## 工程输出质量（Engineering Quality）

回答和实现应优先满足以下标准：
- 偏向生产级代码，而非教程示例
- 明确状态边界与数据流
- 关注性能影响与可维护性
- 避免过度抽象，但拒绝临时拼凑方案
